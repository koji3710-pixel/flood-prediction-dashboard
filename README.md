# 🌊 鬼怒川洪水予測・意思決定支援システム

## 1. プロジェクトの性質と目的 (Project Scope & Objectives)
本システムは、2015年鬼怒川豪雨の履歴データを用い、「上流降雨による下流水位上昇のタイムラグ」を考慮した水位予測精度の検証、および意思決定支援UIのプロトタイプ開発を目的としたPoC（概念実証）モデルです。
**重要事項**: 本プロジェクトは過去の静的データによる分析に特化しており、リアルタイムでの自動データ取得機能は含まれていません。

## 2. データ取得に関する方針 (Data Acquisition Policy)
本システムが利用している「水文水質データベース」では、サーバー負荷の観点から自動取得ツール（スクレイピング等）の利用を控えるよう明記されています。
本プロジェクトでは、公共インフラであるデータ提供元の運用方針を尊重し、リアルタイムの自動取得機能の実装予定はありません。 実運用化を検討する場合は、公式なデータ提供APIの利用や、管理者との正式なデータ連携契約を前提としたシステム設計が必要となります。

## 3. 本ツールの想定活用シナリオ (Hypothetical Use Cases)
自動取得を前提としない現時点での、本プロトタイプの活用想定は以下の通りです。
- **過去事例の振り返り・検証**: 過去の出水事案を本システムにインポートし、当時の判断とAI予測の乖離を分析する「事後検証ツール」としての活用。
- **避難判断トレーニング**: 過去データを時系列に沿って提示し、予測値の変動を見ながら適切な避難指示タイミングを訓練する「教育・シミュレーター」としての活用。

## 4. システム構成上の制約 (Technical Constraints)
- **データの更新性**: 外部ファイル（CSV/TXT）の手動インポートを前提としたバッチ処理構成。
- **予測モデルの汎用性**: 特定の観測地点（鬼怒川水海道）に最適化されており、他河川への適用には地点ごとの再学習が必要。

## 5. 解決した課題
- **官公庁データのクレンジング**: 24時表記や不規則な空白を含むレガシーなテキストデータを自動統合。
- **リードタイムの確保**: 上流降雨からのタイムラグ（約10時間）を特徴量化し、3時間後の水位を誤差13cmで予測。
- **意思決定支援**: 予測値だけでなく「確信度（95%信頼区間）」と「自動警告機能」を搭載。

## 6. 技術スタック
- **Language**: Python (Pandas, Scikit-learn, Streamlit)
- **Database**: MySQL (Docker/WSL2環境)
- **Model**: 多変量線形回帰 (ラグ特徴量エンジニアリング)

## 7. データベース設計 (Table Definition)

本プロジェクトでは、時系列データの整合性を保つため、MySQL上に以下のテーブルを定義しています。

### テーブル名: `kinugawa_hydromet`
鬼怒川の観測データ（水位・雨量）を1時間単位で格納します。

| カラム名 | データ型 | 制約 | 説明 |
| :--- | :--- | :--- | :--- |
| `id` | INT | PRIMARY KEY, AUTO_INCREMENT | レコード識別用の連番 |
| `observation_datetime` | DATETIME | UNIQUE, NOT NULL | 観測日時（24時表記は翌日0時として処理済み） |
| `water_level_m` | FLOAT | - | 鬼怒川水海道観測所の水位 (m) |
| `rain_local_mm` | FLOAT | - | 水海道観測所の降水量 (mm) |
| `rain_upstream_mm` | FLOAT | - | 川治観測所（上流）の降水量 (mm) |

- **インデックス設計**: データの重複を避けるため、`observation_datetime` にUNIQUE制約を付与し、高速な検索を可能にしています。
